# Merge Conflict Practice Repository

## Purpose

This repository is designed specifically for learning and practicing Git merge conflict resolution.

In real software projects, multiple developers often modify the same file or even the same line of code. When Git cannot automatically combine the changes, a merge conflict occurs.

This repository provides a safe environment to understand:

- Branching
- Concurrent edits
- Merge conflicts
- Conflict markers
- Manual conflict resolution

---

## Learning Outcome

After completing this exercise, students should be able to:

- Identify a merge conflict
- Read Git conflict markers
- Resolve conflicting changes manually
- Complete the merge process successfully

---

## Exercise Instructions

1. Create a new branch.
2. Modify the designated line in `shared-file.txt`.
3. Commit and push your changes.
4. Another student will modify the same line.
5. Attempt to merge the changes.
6. Observe the conflict markers generated by Git.
7. Resolve the conflict manually.
8. Commit the resolved version.

---

## Important

- Do not delete files from this repository.
- Only modify the designated practice files.
- The goal is to understand the conflict resolution process, not to avoid conflicts.
